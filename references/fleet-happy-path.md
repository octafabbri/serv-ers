# Fleet Experience: Happy Path — Emergency Tire Service

## What is Serv?

Serv is a voice-first AI dispatcher for roadside tire emergencies. Fleet managers and drivers use it to create a work order by talking — no forms, no hold music. The conversation takes 2–3 minutes. At the end, a PDF work order is generated and ready to dispatch.

**What Serv handles today:**
- Tire, wheel, and mudflap emergencies
- One position per request (call back for additional positions)
- ERS only — same-day, immediate dispatch

**How to start:** Open the app and tap the animated orb.

---

## Happy Path: Fleet Manager Calling in for a Driver

**Scenario:** A fleet manager is at the office. One of their drivers, Tom Bradley, has a blown front steer tire on I-80 in Wyoming. The manager opens Serv on their device and calls it in.

### Step-by-step flow

| Step | What happens |
|------|-------------|
| 1 | Manager taps the orb — Serv greets and states its scope |
| 2 | Serv asks: driver or fleet manager? |
| 3 | Manager identifies themselves — Serv collects manager name + phone |
| 4 | Serv asks for driver name + phone |
| 5 | Serv asks for fleet name and Ship To |
| 6 | Serv asks truck or trailer, unit number, VIN |
| 7 | Serv asks for location |
| 8 | Serv asks what's needed at the position (replace/repair, wheel, mudflap, or combination) |
| 9 | Serv collects tire details (condition, brand/size, position) |
| 10 | Urgency auto-set to ERS (always in this flow) |
| 11 | Serv reads back the full summary |
| 12 | Manager says yes — work order is generated |
| 13 | PDF is ready to download |

## Conversation Script

```
[Manager opens app, taps the orb]

SERV:  "Good afternoon! I'm Serv, your Fleet Services AI Assistant.
        At this moment, I handle tire, wheel, and mudflap emergencies —
        one position per request. What do you need?"

MANAGER: "Hi, I've got a driver out on the highway with a blown tire.
          Need to get someone out there."

SERV:  "Copy that. First — are you the driver on the truck, or are you
        a fleet manager calling on behalf of a driver?"

MANAGER: "I'm the fleet manager, calling for one of my drivers."

SERV:  "Got it. What's your name and phone number?"

MANAGER: "Michael Chen, 555-0147."

SERV:  "Thanks Michael. What's the driver's name and phone number?"

MANAGER: "Tom Bradley, 555-0198."

SERV:  "Got it. What's your fleet name?"

MANAGER: "Premier Logistics."

SERV:  "What's the Ship To for your fleet?"

MANAGER: "PREMIER_DFW01."

SERV:  "Is this for a truck or a trailer?"

MANAGER: "It's a truck."

SERV:  "What's the unit number?"

MANAGER: "Unit 47."

SERV:  "And the VIN?"

MANAGER: "1HGBH41JXMN109186."

SERV:  "Where's the truck right now?"

MANAGER: "He's on I-80, mile marker 245, east of Laramie, Wyoming.
          Pulled over on the shoulder."

SERV:  "What do you need at that position — a tire replaced or repaired,
        a wheel, a mudflap, or a combination?"

MANAGER: "Tire replaced. It blew out completely."

SERV:  "Understood. What's the condition — blown out, flat, leaking air,
        sidewall cut, or something else?"

MANAGER: "Blown out."

SERV:  "What size or brand?"

MANAGER: "Michelin XDA, 295/75R22.5."

SERV:  "Which position?"

MANAGER: "Left front steer."

SERV:  "Alright, let me read this back to you.
        Driver name, Tom Bradley. Phone, 555-0198.
        Fleet, Premier Logistics.
        Location, I-80 mile marker 245 east of Laramie Wyoming.
        Vehicle type, truck.
        Service type, tire replacement. Tire, Michelin XDA 295/75R22.5.
        Quantity, 1. Position, left front steer.
        Priority, emergency same-day.
        Say yes to submit, or tell me what needs to change."

MANAGER: "That's all correct. Yes, go ahead."

SERV:  "Got it, your work order is ready to download."

[UI transitions to pdf-ready state]
[Red urgency banner: EMERGENCY ROAD SERVICE (TODAY/SAME-DAY)]
[Download button]
```

## What Happens Next (Provider Side)

Once submitted, the work order is visible to service providers on the platform. The provider can:

- **Accept** — confirms the job as dispatched
- **Decline** — with optional reason; fleet is notified
- **Counter-propose** — suggest a different date/time if ERS is not available

> **Note:** Counter-proposal review (voice and UI overlays) is currently disabled. When a counter-proposal comes in, the fleet manager sees a notification in the requests list but cannot respond through the app at this time.

## Edge Cases in This Flow

| Situation | How Serv handles it |
|-----------|-------------------|
| Multiple blown tires | Captures the most critical position; prompts manager to call back for the others |
| Manager unsure of tire size | Serv still completes the request; size can be noted as unknown |
| Driver calls in directly (not fleet manager) | Serv skips the "caller" fields and collects driver info directly |
| Vague location ("somewhere in Wyoming") | Serv asks for more specifics — highway, mile marker, exit, or nearest landmark |
| Manager says "schedule for tomorrow" | Serv redirects: this line is ERS only; confirms to proceed as emergency or advises main dispatch line |
| VIN is unknown | Unit number is automatically used as the VIN fallback — no re-asking needed |
| Ship To is not provided | Fleet name is used as the Ship To fallback automatically |
